1. 需求背景

1.1 业务背景

Hunter 系统是 OFW 信贷获客系统，需要管理多个 AI Agent 协同完成获客、风控、贷后等业务流程。当前缺乏统一的 Agent 管理平台，导致：

  • Agent 分散管理，无法统一调度
  • 缺乏状态监控，故障发现滞后
  • 无法编排多 Agent 协同工作流
  • 新 Agent 接入成本高

1.2 用户痛点

用户角色痛点影响
运营人员无法实时查看 Agent 状态故障发现滞后，影响业务
技术人员新 Agent 接入流程复杂开发成本高，周期长
产品经理无法灵活编排业务流程业务迭代慢，响应市场慢
管理层缺乏 Agent 效能数据无法优化资源配置

1.3 预期收益

收益维度预期效果衡量指标
效率提升Agent 接入时间从 3 天降至 1 小时接入时长
稳定性故障发现时间从小时级降至秒级MTTR
业务灵活性新业务流程编排从周级降至天级上线周期
资源优化Agent 利用率提升 30%资源利用率

目标：构建统一的 Agent 管理平台，实现 Agent 注册与发现、状态监控与心跳检测、多 Agent 工作流编排、执行日志与效能分析。

7. 验收标准

P0 核心功能

  • AC001 Agent 注册：可以成功注册 A2A/MCP/Local 三种类型 Agent
  • AC002 Agent 列表：可以查询 Agent 列表，支持筛选
  • AC003 心跳检测：30 秒心跳，90 秒超时自动下线
  • AC004 工作流创建：可以创建 YAML 配置工作流
  • AC005 工作流执行：可以执行工作流，返回 execution_id
  • AC006 状态查询：可以查询执行状态和进度
  • AC007 变量传递：上一步输出可以传递给下一步
  • AC015 单元测试：覆盖率 >80%

P1 增强功能

  • AC008 条件分支：可以根据条件跳过步骤
  • AC009 重试机制：失败步骤可以自动重试
  • AC010 日志记录：记录完整执行日志
  • AC011 权限控制：不同角色权限隔离
  • AC013 异常告警：Agent 离线时发送告警通知
  • AC014 API 文档：完整的 OpenAPI 文档

P2 扩展功能

  • AC012 数据看板：显示 Agent 统计和效能分析

| Agent | 职责 | 决策权 | 记忆内容 | 独立权力 |
|------|------|--------|---------|---------|
| **总调度** | 任务分发、状态同步、冲突仲裁 | 战略：打哪个痛点、优先级 | 任务历史、Agent 表现 | 无（不干预战术） |
| **情报 Agent** | 采集、分析、清洗 | 战术：采哪些、过滤规则 | 评论库、竞品历史、关键词效果 | 拒绝任务、申请资源 |
| **执行 Agent** | 发言、投放、追踪 | 战术：用哪个账号、何时执行 | 执行记录、转化数据、账号表现 | 拒绝任务、申请资源 |
| **风控 Agent** | 检测、判断、告警 | 战术：是否暂停、是否切换 | 风险事件、封号记录、阈值调整 | 紧急暂停、建议策略 |

**核心原则：总调度是任务分发平台，Agent 是独立作战单位。**

## 四、核心流程

### 4.1 任务执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      任务执行流程                                │
└─────────────────────────────────────────────────────────────────┘

① 总调度创建任务
   {type: "post", pain_point: "放款慢", channel: "FB"}
   │
   ▼
② 分配 Agent
   → 执行 Agent
   │
   ▼
③ Agent 加载剧本
   → 发言剧本 v2.0（策略 + 配置 + 资源引用）
   │
   ▼
④ Agent 自主选择资源
   → 账号 A（人设：过来人，状态：正常）
   │
   ▼
⑤ 执行任务
   → 调用 API 发言
   │
   ▼
⑥ 生成执行记录
   → 记录内容、结果、时间
   │
   ▼
⑦ 数据回流
   → Agent 记忆更新 + 剧本消耗标记 + 资源使用记录
```

## 七、核心设计原则

| 原则 | 说明 | 33 制度映射 |
|------|------|------------|
| **Agent 是能力 + 记忆的集合** | 持久化，越用越聪明 | 智能体为最小作战单位 |
| **剧本是策略 + 配置 + 资源的集合** | 消耗品，用了要变体 | 战术方法论 |
| **执行记录是数据回流的枢纽** | 没有它，飞轮转不起来 | 经验积累 |
| **总调度管战略，Agent 管战术** | 决策分层，扁平高效 | 战略 - 战术分离 |
| **马甲是消耗品，Agent 永生** | 账号封了换号，Agent 永不阵亡 | 作战单位独立 |
| **权责对等** | Agent 有独立权力，也要独立担责 | 最小作战单位核心 |

## 八、关键机制设计（MVP 版）

### 8.1 Agent 记忆机制

| 设计点 | MVP 方案 | 扩展方向 |
|--------|---------|---------|
| **更新频率** | 每次执行后更新 | 批量更新（性能优化） |
| **更新内容** | 执行结果 + 关键指标 | 情境 - 决策 - 结果三元组 |
| **记忆上限** | 最近 50 条 | 基于效果淘汰（差决策先淘汰） |
| **淘汰策略** | FIFO（先进先出） | 效果差的优先淘汰 |
| **冷启动** | 前 10 次用保守配置 | 预置行业经验库 |
| **记忆共享** | 无（Agent 独立） | Agent 之间可查询彼此经验 |

**MVP 原则：先跑起来，后续可扩展为结构化经验库。**

---

### 8.2 Agent 权责对等

| 权力 | 当前 | MVP 方案 | 责任 |
|------|------|---------|------|
| **任务否决权** | ❌ 无 | ✅ 资源不足时可拒绝 | 对拒绝原因负责 |
| **资源请求权** | ❌ 无 | ✅ 主动申请新账号/话术 | 对资源使用效果负责 |
| **策略建议权** | ❌ 无 | ✅ 可建议剧本变体 | 对建议效果负责 |
| **紧急处置权** | ❌ 无 | ✅ 风控可紧急暂停 | 对处置后果负责 |

**核心原则：有权力就要有责任，权责对等。**

---

### 8.3 资源锁机制

| 场景 | MVP 方案 | 说明 |
|------|---------|------|
| **账号抢占** | 简单锁（占用标记） | Agent 取号时标记"使用中" |
| **状态同步** | 执行后更新状态 | 风控检测后同步给总调度 |
| **冲突处理** | 风控优先 | 风控标记封号，执行 Agent 立即停用 |

---

### 8.4 风控处置权

| 情况 | 风控 Agent 能做什么 | 需要总调度批准 |
|------|-------------------|--------------|
| 单账号异常 | 暂停该账号使用 | ❌ |
| 封号率>10% | 告警 + 建议暂停 | ✅ |
| 批量封号 | 紧急暂停任务 | ❌（先执行后汇报） |
| 切换账号池 | 自动切换备用池 | ❌ |

**原则：小问题自动处理，大问题告警 + 建议。**

---

### 8.5 总调度边界（MVP）

| 总调度能做 | 总调度不能做 |
|-----------|-------------|
| ✅ 分发任务给 Agent | ❌ 指定 Agent 用哪个账号 |
| ✅ 同步 Agent 状态（只读） | ❌ 干预 Agent 战术决策 |
| ✅ 仲裁 Agent 冲突（仅当请求） | ❌ 直接调用剧本 |
| ✅ 统计任务完成情况 | ❌ 修改 Agent 记忆 |

**核心：总调度是任务分发平台，不是指挥中心。**

## 九、决策权分层

### 9.1 三层决策架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      决策权分层                                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────┬─────────────────┬─────────────────────────────┐
│   总调度        │     Agent       │        剧本                 │
│  (战略决策)     │   (战术决策)    │      (执行决策)             │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ • 打哪个痛点    │ • 用哪个剧本    │ • 用哪个账号                │
│ • 预算多少      │ • 参数怎么调    │ • 什么时候执行              │
│ • 优先级排序    │ • 资源怎么选    │ • 话术变体选哪个            │
│ • 启停控制      │ • 失败怎么处理  │ • 执行频率控制              │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

| 层级 | 决策者 | 决策类型 | 持久性 | 示例 |
|------|--------|---------|--------|------|

### 9.4 决策冲突处理

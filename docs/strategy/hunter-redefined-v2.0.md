# 🎯 Hunter 系统 · 重新定义

> **定位：** 用户注意力的截流与转化引擎  
> **核心价值：** 在用户注意力出现的地方，主动触达并转化为借贷意向  
> **核心指标：** 触达→申请转化率 >15%

---

## 一、本质思考

### 1.1 什么是"用户注意力截流"？

```
传统获客：用户主动搜索 → 看到广告 → 点击申请
    特征：被动等待，获客成本$4.5/人

Hunter 截流：用户表达需求 → Agent 主动触达 → 引导申请
    特征：主动截流，获客成本$0.3-1/人
```

**注意力的本质：用户在哪里表达借贷需求，Hunter 就在哪里出现。**

| 场景 | 用户注意力表现 | Hunter 截流方式 |
|------|--------------|--------------|
| FB 借贷社群 | 用户发帖"急需借钱" | Agent 私信触达 |
| FB 评论区 | 用户评论"我也需要" | Agent 回复 + 私信 |
| KOL 内容下 | 用户评论"怎么申请" | Agent 引导私信 |
| WhatsApp 群 | 用户询问借贷信息 | Agent 自动回复 |

---

### 1.2 什么是"转化引擎"？

**转化 = 意图识别 × 信任建立 × 行动引导**

```
用户表达需求 → 意图识别Agent → 知识库Agent → 申请Agent → 完成申请
    ↓              ↓              ↓              ↓
  注意力        理解需求        解答疑问        引导行动
```

**引擎的本质：自动化、规模化、可优化。**

| 维度 | 人工转化 | Hunter 引擎转化 |
|------|---------|--------------|
| 响应速度 | 小时级 | 秒级 |
| 并发能力 | 1 对 1 | 1 对 1000+ |
| 转化率 | 5-10% | 目标 15%+ |
| 可优化性 | 依赖经验 | 数据驱动 A/B 测试 |

│  ├─────────────────────┤    ├─────────────────────┤        │

│  ├─────────────────────┤    ├─────────────────────┤        │

│  │ - 意图识别 Agent     │    │ - Agent 心跳检测    │        │

## 四、Hunter 核心指标

### 4.1 北极星指标

**触达→申请转化率 >15%**

### 4.2 过程指标

| 指标 | 定义 | 健康值 | 警戒值 |
|------|------|--------|--------|
| 注意力捕捉量 | 每日发现的需求帖/评论数 | >500/日 | <200/日 |
| 触达率 | 成功触达用户比例 | >80% | <50% |
| 意向转化率 | 触达→意向用户比例 | >30% | <15% |
| 申请转化率 | 意向→申请用户比例 | >50% | <30% |
| 获客成本 | 单个申请成本 | <$1 | >$2 |

### 4.3 结果指标

| 指标 | 目标值 | 验证方式 |
|------|--------|---------|
| 月获客数 | 2000-5000 人 | 申请数统计 |
| 获客成本 | $0.3-1/人 | 总成本/获客数 |
| 转化率 | >15% | 触达→申请漏斗 |
| 封号率 | <5%/月 | 账号存活监控 |

## 八、核心结论

### 8.1 Hunter 的本质

**不是 Agent 管理平台，是用户注意力截流与转化引擎。**

### 8.2 Hunter 的边界

| 做什么 | 不做什么 |
|--------|---------|
| FB 社群/评论区截流 | SEO 内容引流（独立） |
| 意图识别 + 转化引导 | KOL 商务合作（独立） |
| 申请流程引导 | 风控审批（独立） |
| 转化数据优化 | 贷后管理（独立） |

### 8.3 Hunter 的成功标准

**用最低成本（<$1/人），截获最多注意力（日均 500+），转化最多申请（转化率>15%）。**

# Hunter 功能信息结构分析

> 本文档分析 Hunter Agent 管理系统的信息架构（Information Architecture），识别结构问题并提出改进建议。

---

## 一、当前信息结构总览

### 1.1 导航层级

```
Hunter (系统名称)
├── 📊 Dashboard      → 系统概览、数据统计
├── 🤖 Agent 管理     → Agent 创建、配置、监控
├── ⚡ Skill 管理     → Skill 定义、绑定
├── 📝 执行记录       → 任务执行历史、日志
└── ⚙️ 系统设置       → 配置、API 密钥
```

**层级深度：2 级（扁平结构）**
- 一级：主导航（5 个入口）
- 二级：页面内功能模块

---

### 1.2 各页面功能模块

#### 📊 Dashboard
| 模块 | 功能 | 数据来源 |
|------|------|---------|
| **统计卡片** | 总执行次数、成功率、平均耗时、失败任务 | 执行记录聚合 |
| **趋势图表** | 近 7 天执行趋势（折线图） | 执行记录聚合 |
| **活跃 Agent** | 当前活跃 Agent 快速入口 | Agent 状态 |
| **失败任务快览** | 最近失败任务列表（点击跳转） | 执行记录筛选 |

#### 🤖 Agent 管理
| 模块 | 功能 | 关联数据 |
|------|------|---------|
| **统计卡片** | 总 Agent、Online、Running、Offline | Agent 聚合 |
| **搜索筛选** | 搜索 Agent 名称/ID | 无 |
| **Agent 卡片** | 状态、绑定 Skills、最近执行次数 | Skill 关联、执行记录关联 |
| **创建 Agent** | 自然语言创建 → 配置确认 | Skill 选择 |

**Agent 卡片操作：**
- 📋 详情 → Agent 详情 Drawer
- ⚙️ 配置 → 配置 Drawer（触发方式、间隔、Skills）
- 📝 日志 → 运行日志 Drawer
- ⏸️暂停/⏹️停止/🔄重启 → 状态操作

#### ⚡ Skill 管理
| 模块 | 功能 | 关联数据 |
|------|------|---------|
| **统计卡片** | 总 Skills、Active、Inactive、本周新增 | Skill 聚合 |
| **Skill 表格** | 名称、描述、状态、绑定 Agent 数、操作 | Agent 关联 |
| **创建 Skill** | 名称、描述、输入输出定义 | 无 |

**Skill 表格操作：**
- ✏️ 编辑 → 编辑 Skill Drawer
- 🗑️ 删除 → 删除确认

#### 📝 执行记录
| 模块 | 功能 | 关联数据 |
|------|------|---------|
| **统计卡片** | 总执行次数、成功率、平均耗时、失败任务 | 执行记录聚合 |
| **搜索筛选** | 搜索 + 状态筛选（成功/运行中/失败） | 无 |
| **执行表格** | ID、Agent、状态、耗时、时间、操作 | Agent 关联 |
| **详情 Drawer** | 基本信息、执行日志、结果数据 | Agent、日志 |

**执行记录操作：**
- 📋 详情 → 执行详情 Drawer
- 🔄 重试（失败任务） → 重试执行
- ⏹ 取消（运行中任务） → 取消执行

**Drawer Footer 按钮：**
- 关闭 | 🤖查看 Agent | 📝完整日志 | 📋复制 ID
- 失败：+ 🔄重试 + 🗑️删除
- 运行中：+ ⏹取消

#### ⚙️ 系统设置
| 模块 | 功能 | 影响范围 |
|------|------|---------|
| **通用设置** | 通知、自动刷新、声音提示 | 全系统 |
| **Agent 配置** | Heartbeat、超时、重试次数 | Agent 行为 |
| **API 密钥** | OpenAI、Meta、WhatsApp 密钥 | Agent 执行能力 |
| **数据库连接** | Connection String、连接池 | 数据存储 |
| **危险操作** | 清空执行记录、重置系统 | ⚠️ 高风险 |

## 二、核心数据实体与关系

### 2.1 实体模型

```
┌─────────────────────────────────────────────────────────────┐
│                         Agent                                │
│  ID: AGENT-001                                               │
│  名称: 获客流程 Agent                                         │
│  状态: Online | Running | Offline                            │
│  绑定 Skills: [Skill-1, Skill-2, Skill-3]                   │
│  配置: 触发方式、间隔、超时、重试                              │
│  创建时间: 2026-03-15                                        │
└─────────────────────────────────────────────────────────────┘
           │
           │ 1:N (一个 Agent 执行多次)
           ▼
┌─────────────────────────────────────────────────────────────┐
│                      执行记录 (Execution)                     │
│  ID: EXEC-20260416-001                                       │
│  Agent: AGENT-001                                            │
│  状态: Success | Running | Failed                            │
│  开始时间: 2026-04-16 10:23                                   │
│  耗时: 2.3s                                                   │
│  日志: [INFO, INFO, SUCCESS...]                              │
│  结果: {触达用户数: 3, 转化意向: 2}                           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                         Skill                                │
│  ID: SKILL-001                                               │
│  名称: FB 社群监控                                           │
│  描述: 监控 Facebook 群组帖子                                 │
│  输入: 群组 ID、关键词                                        │
│  输出: 潜在意向帖子列表                                       │
│  状态: Active | Inactive                                     │
│  绑定 Agent 数: 5                                            │
└─────────────────────────────────────────────────────────────┘
           │
           │ N:N (多个 Skill 可绑定多个 Agent)
           ▼
        Agent.bindingSkills[]
```

## 三、跨页面导航流向

### 3.1 正向导航（用户主动操作）

```
Dashboard
    │
    ├── 点击"失败任务" → 执行记录（自动筛选失败）
    │
    └── 点击"活跃 Agent" → Agent 管理
    
Agent 管理
    │
    ├── 点击 Agent 详情 → 查看 Skills 绑定
    │
    └── 点击日志 → 查看运行日志
    │
    └── 点击 Skill 名称 → Skill 管理（❌ 未实现）
    
Skill 管理
    │
    ├── 点击"绑定 Agent 数" → Agent 管理（筛选该 Skill）（❌ 未实现）
    │
    └── 点击编辑 → 编辑 Skill
    
执行记录
    │
    ├── 点击 Agent 名称 → Agent 详情（❌ 未实现跳转）
    │
    ├── 点击"查看 Agent" → Agent 管理（Drawer 按钮）
    │
    └── 点击重试 → 重新执行任务
    
系统设置
    │
    └── 修改配置 → 影响 Agent 行为
```

### 3.2 逆向导航（数据溯源）

```
执行记录
    │
    └── 我属于哪个 Agent？ → Drawer "查看 Agent" 按钮
    
Agent
    │
    └── 我绑定了哪些 Skills？ → 详情 Drawer 显示
    │
    └── 我最近的执行记录？ → 详情 Drawer 显示最近 5 条
    
Skill
    │
    └── 哪些 Agent 用了我？ → 表格显示"绑定 Agent 数"
    │
    └── 我参与的执行记录？ → ❌ 未实现
```

## 四、问题识别与分析

### 4.1 🔴 严重问题（影响核心功能）

| 问题 | 影响 | 页面 | 建议 |
|------|------|------|------|
| **Agent → Skill 跳转缺失** | Agent 详情中点击 Skill 无法跳转到 Skill 管理 | Agent | 添加 Skill 可点击跳转 |
| **Skill → Agent 跳转缺失** | Skill 表格"绑定 Agent 数"无法点击跳转 | Skill | 添加数字可点击，筛选 Agent |
| **执行记录 → Agent 跳转不完整** | 表格中 Agent 名称无法点击跳转 | Executions | 添加 Agent 名称可点击 |
| **Skill 执行记录缺失** | 无法查看某 Skill 的执行历史 | Skill | 添加"执行记录"按钮或 Drawer |
| **Dashboard 失败任务点击无反馈** | 点击失败任务卡片无跳转 | Dashboard | 添加点击跳转逻辑 |

### 4.2 🟡 中等问题（影响体验）

| 问题 | 影响 | 页面 | 建议 |
|------|------|------|------|
| **筛选结果无持久化** | 刷新页面筛选条件丢失 | Executions/Agents | URL 参数保存筛选状态 |
| **无全局搜索** | 无法跨实体搜索 | 全局 | 添加顶部全局搜索框 |
| **Drawer 内 Skill 不可点击** | Agent 详情 Drawer 内 Skills 无法交互 | Agent Drawer | Skills 可点击跳转 |
| **创建 Skill 功能不完整** | 点击创建 Skill 无交互 | Skill | 添加创建 Skill Drawer |
| **执行日志筛选无实际效果** | 日志级别筛选下拉框无功能 | Executions Drawer | 实现日志筛选功能 |

### 4.3 🟢 小问题（优化建议）

| 问题 | 影响 | 页面 | 建议 |
|------|------|------|------|
| **统计卡片点击无反馈** | 统计卡片点击无交互 | Dashboard/Agents | 添加 Hover 提示或点击跳转 |
| **时间显示格式不统一** | PHT/UTC 混用可能困惑 | 多页面 | 统一显示 PHT + UTC 双时区 |
| **Agent 状态操作无确认** | 暂停/停止/删除无确认弹窗 | Agent | 添加确认 Modal |
| **Drawer 无快捷键提示** | 用户不知道 ESC 可关闭 | 全局 Drawer | Footer 提示"按 ESC 关闭" |

## 五、信息结构改进建议

### 5.1 实体链接矩阵（必须实现）

| 从实体 | 目标实体 | 触发方式 | 当前状态 | 建议实现 |
|--------|---------|---------|---------|---------|
| **执行记录** → Agent | 点击 Agent 名称 | ❌ 未实现 | ✅ 必须 |
| **Agent** → Skill | 详情 Drawer 点击 Skill 标签 | ❌ 未实现 | ✅ 必须 |
| **Skill** → Agent | 点击"绑定 Agent 数" | ❌ 未实现 | ✅ 必须 |
| **Skill** → 执行记录 | 添加"执行记录"按钮 | ❌ 未实现 | ✅ 必须 |
| **Dashboard** → 执行记录 | 点击失败任务卡片 | ❌ 未实现 | ✅ 必须 |
| **Dashboard** → Agent | 点击活跃 Agent 卡片 | ❌ 未实现 | ✅ 必须 |

### 5.2 导航层级优化建议

**当前：扁平 2 级**
```
Dashboard | Agents | Skills | Executions | Settings
```

**建议：增加子导航（可选）**

```
Agent 管理
├── 全部 Agent
├── Online Agent
├── Running Agent
└── Offline Agent

执行记录
├── 全部记录
├── 成功记录
├── 运行中
└── 失败记录
```

**实现方式：**
- 不增加物理导航层级
- 通过筛选 Tab 或 URL 参数实现子视图
- 统计卡片点击直接跳转到筛选视图

### 5.3 全局搜索建议

```
┌─────────────────────────────────────────────────────┐
│ 🔍 搜索 Agent、Skill、执行记录...                    │
└─────────────────────────────────────────────────────┘
         │
         ├── 输入 "AGENT-001" → Agent 详情 Drawer
         ├── 输入 "EXEC-2026" → 执行记录 Drawer
         ├── 输入 "获客" → Agent 列表筛选
         └── 输入 "FB 社群" → Skill 列表筛选
```

## 二、核心决策：执行记录的对象选择

### 问题：执行记录的对象应该是 Agent 还是 Skill？

这是一个架构设计的核心问题，需要从业务本质分析。

### 业务场景分析

根据 OFW 顶层设计，**Agent 是流程编排者，Skill 是能力单元**：

```
获客 Agent (AGENT-001)
    │
    ├── FB 社群监控 Skill → 发现帖子
    ├── 意图识别 Skill → 判断需求
    └── 私信触达 Skill → 发送消息
    
风控 Agent (AGENT-003)
    │
    ├── 身份验证 Skill
    ├── 信用评分 Skill
    └── 决策引擎 Skill
```

**用户关心的两个层级：**

| 层级 | 用户问题 | 对应实体 |
|------|---------|---------|
| **流程级** | "获客 Agent 今天执行了几次？成功率？" | Agent 执行记录 |
| **能力级** | "FB 监控 Skill 发现了多少帖子？" | Skill 执行记录 |

### 三种方案对比

| 维度 | 方案 A：纯 Agent | 方案 B：纯 Skill | 方案 C：两层结构 |
|------|-----------------|-----------------|-----------------|
| **执行粒度** | 任务级（一次完整流程） | 能力级（一次具体操作） | 两者都有 |
| **记录数量** | 少（Agent 执行次数） | 多（Skill 执行次数 × N） | Agent + 子 Skill |
| **用户视角** | "获客流程执行了" | "FB 监控执行了" | 两者都能 |
| **溯源能力** | 知道哪个 Agent 在工作 | 知道哪个 Skill 在工作 | 都能追溯 |
| **数据归属** | 执行 → Agent | 执行 → Skill | 执行 → Agent → Skills |
| **流程完整性** | ✅ 保持 | ❌ 丢失（看不到整体） | ✅ 保持 |
| **细粒度追溯** | ❌ 缺失 | ✅ 完整 | ✅ 完整 |
| **数据量** | 合理 | 爆炸（1 Agent = 3-5 Skill） | 合理 |
| **拓展性** | 一般 | Skill 页面独立但流程断裂 | 最佳 |

### 纯 Skill 执行记录的问题

| 问题 | 说明 |
|------|------|
| **流程完整性丢失** | 无法知道"获客流程整体是否成功" |
| **数据量爆炸** | 一条 Agent 执行会产生 3-5 条 Skill 记录 |
| **用户认知负担** | 用户关心的是"任务完成"而非"每个能力执行" |
| **聚合困难** | 想看 Agent 整体表现需要手动聚合多条 Skill |

### 纯 Agent 执行记录的问题

| 问题 | 说明 |
|------|------|
| **内部细节丢失** | 无法查看 Agent 内部各 Skill 的执行细节 |
| **无法定位失败点** | Agent 失败了，不知道是哪个 Skill 出问题 |
| **Skill 溯源缺失** | 无法从执行记录跳转到 Skill 详情 |

### 最终决策：方案 C（两层结构）

**理由：**

| 维度 | 说明 |
|------|------|
| **业务准确性** | 还原真实结构：Agent 执行 = 多个 Skill 协作 |
| **用户需求** | 用户既关心整体（Agent），也关心细节（Skill） |
| **拓展性** | Agent 页面 + Skill 页面各自独立，互不干扰 |
| **数据量** | 合理，主记录为 Agent，子记录为 Skill |
| **原型阶段** | 验证"两层感"，正式开发时再完整实现数据结构 |

## 三、执行记录结构改进方案

### 来源：IA_ANALYSIS.md + 两层执行记录讨论

#### 当前问题

| 问题 | 说明 |
|------|------|
| **信息结构不完整** | 执行记录只显示 Agent，未展示内部 Skill 调用 |
| **业务本质被掩盖** | Agent 执行 = 多个 Skill 协作，但用户看不到 |
| **溯源能力缺失** | 无法从执行记录跳转到 Skill 详情 |

#### 方案对比（美化与拓展性）

| 维度 | 方案 A：增强 Agent Drawer | 方案 B：两层执行记录 | 推荐 |
|------|--------------------------|---------------------|------|
| **信息层次** | 挤在一个 Drawer | Agent 主、Skill 子 | B ✅ |
| **美感** | Drawer 臃肿 | 分层展示，层次分明 | B ✅ |
| **拓展性** | 拆分需重构 | Skill 页面天然独立 | B ✅ |
| **业务准确性** | 掩盖调用关系 | 还原真实结构 | B ✅ |
| **原型工作量** | 小（1-2h） | 中（3-4h） | A 略省 |

#### 最终选择：方案 B（两层执行记录）

**理由：原型阶段要验证的信息结构，两层更接近真实业务，拓展性更好，美感更佳。**

---

### 方案 B 具体改动

#### 1. 执行记录表格新增列

```diff
| ID | Agent | 状态 | 耗时 | 时间 | 操作
+ | ID | Agent | Skills 调用 | 状态 | 耗时 | 时间 | 操作
```

**Skills 调用列显示方式：**

| 场景 | 显示 |
|------|------|
| 成功 | `3 Skills ✓` 或 `FB监控 → 意图识别 → 私信触达` |
| 部分失败 | `3 Skills (1失败)` 或最后一项显示 `✕` |
| 运行中 | `正在进行 (第2个)` |

#### 2. Agent 详情 Drawer 增加 Skill 调用区

```
┌─────────────────────────────────────────────────────────────┐
│ Drawer Header: EXEC-20260416-001                            │
├─────────────────────────────────────────────────────────────┤
│ 基本信息                                                     │
│   Agent: AGENT-001 (获客流程)                                │
│   状态: ✓ 成功                                               │
│   总耗时: 2.3s                                               │
├─────────────────────────────────────────────────────────────┤
│ Skill 调用明细 ← 新增 Section                                │
│   ┌───────────────────────────────────────────────────────┐ │
│   │ ✓ FB 社群监控        0.8s    [点击查看详情]            │ │
│   │ ✓ 意图识别           0.5s    [点击查看详情]            │ │
│   │ ✓ 私信触达           1.0s    [点击查看详情]            │ │
│   └───────────────────────────────────────────────────────┘ │
│   （失败时：✕ 私信触达  1.0s  红色高亮）                     │
├─────────────────────────────────────────────────────────────┤
│ 执行结果                                                     │
│   触达用户数: 3                                              │
│   转化意向: 2                                                │
├─────────────────────────────────────────────────────────────┤
│ Footer: 关闭 | 查看完整日志 | 查看关联 Agent                 │
└─────────────────────────────────────────────────────────────┘
```

#### 3. Skill 调用卡片可点击

点击后打开 Skill 详情 Drawer：

```
┌─────────────────────────────────────┐
│ Skill Execution Drawer              │
│ - Skill 名称: FB 社群监控            │
│ - 执行状态: ✓ 成功                   │
│ - 耗时: 0.8s                        │
│ - 输入: 群组 ID、关键词              │
│ - 输出: 发现 3 个潜在意向帖子        │
│ - Skill 详情日志                     │
└─────────────────────────────────────┘
```

#### 4. Skill 页面增加执行入口

```
Skill 管理
├── 详情 Drawer
│   ├── 基本信息
│   ├── 输入输出定义
│   ├── 绑定 Agent（可点击跳转）
+   │   ├── 本月执行次数: 48
+   │   ├── 成功率: 95.8%
+   │   ├── 平均耗时: 0.7s
+   │   └── [查看执行记录] 按钮 → 跳转执行记录筛选页
│   └── 删除（需确认）
```
